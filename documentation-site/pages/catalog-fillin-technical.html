Technical Implementation

A deep dive into FillInOntologyObject.java, property discovery, and targeted AI extraction logic.

Implementation

The FillInOntologyObject Class

The FillInOntologyObject service is responsible for populating complex scientific objects with values extracted from natural language prompts.

Data Structure: FillInResult

The result of an extraction is split into assigned pairs and unassigned slots, allowing the UI to highlight missing data.

public static class FillInResult {
    private Map<String, String> assignedProperties; // Successfully extracted
    private List<String> unassignedProperties;     // Expected but not found
}
Stage 1: Discovery

Dynamic Property Discovery

Before calling the AI, the system uses DataTermsUtilities to discover the "Property Set" of the target class. This ensures the extraction is grounded in the current version of the ontology. 

// 1. Get the list of simple property classnames
List<String> expectedProperties = DataTermsUtilities.getSimpleProperties(classname);
Stage 2: Prompting

Targeted Preamble Generation

The system constructs a dynamic PREAMBLE that injects the expectedProperties as context. This explicitly tells the LLM exactly which keys it is allowed to use in its output. 

String propertyContext = String.join(", ", expectedProperties);
String preamble = String.format("""
    ROLE: You are an Extraction Agent.
    CONTEXT: You are looking for values for properties: [%s]
    TASK: Analyze USER QUERY and extract these specific classnames.
    FORMAT: Return ONLY a JSON object.
    """, propertyContext);
Stage 3: Integration

Discovery Engine Orchestration

The extraction leverages the SearchServiceClient to perform a grounded summarization. 

SearchRequest request = SearchRequest.newBuilder()
    .setServingConfig(servingConfig)
    .setQuery(prompt)
    .setFilter(filter) // Ground the extraction in scientific context
    .setContentSearchSpec(SearchRequest.ContentSearchSpec.newBuilder()
        .setSummarySpec(SearchRequest.ContentSearchSpec.SummarySpec.newBuilder()
            .setModelPromptSpec(
                SearchRequest.ContentSearchSpec.SummarySpec.ModelPromptSpec.newBuilder()
                    .setPreamble(preamble)
                    .build())
            .build())
        .build())
    .build();
Stage 4: Post-Processing

Gap Analysis and Verification

The final stage uses Java Streams to compare the AI's output with the original "Expected" list to identify what was missed. 

// Identify unassigned properties
List<String> unassigned = expectedProperties.stream()
        .filter(prop -> !assigned.containsKey(prop))
        .collect(Collectors.toList());

return new FillInResult(assigned, unassigned);

Low Level Overview

Automated Form Filling for Scientists

Scientific data objects (Catalog Objects) can be complex, often requiring dozens of specific fields such as titles, URLs, descriptions, and methodology. The Fill In Catalog Object system is a specialized AI agent that "reads" a scientist's experimental notes and automatically fills out the corresponding catalog entry.

How it Works

Imagine you have an empty digital form for a "Data Source". Instead of copying and pasting values one by one, you paste your entire project abstract. The agent identifies the Title, the author's URL, and the core description, mapping them directly to the form fields with high precision.

Higher Level Overview

The "Contract of Properties"

Unlike general-purpose extraction, this system is targeted. It doesn't just look for anything interesting; it looks specifically for the fields defined by the ontology.

1. Dynamic Schema Discovery

The system first queries the scientific ontology to find exactly which properties are "expected" for a specific class (e.g., a dataset:ActivityRepositoryInitialReadLocalFile).

2. Targeted Prompting

This list of expected fields is inserted into the AI's instructions. This creates a strict contract: "Only find values for these specific parameters."

3. Gap Analysis

After extraction, the system performs a logic check. It clearly distinguishes between what it was able to Assign and what remains Unassigned, allowing the user to provide missing details manually if needed.

Low Level Overview

Bridging Language and Logic

When entering scientific data, researchers often describe their experiments in natural language. The Value and Term Extraction system automates the tedious task of mapping these descriptions to specific database fields. For example, if a researcher writes "The pressure was recorded at 1.5 atm," the system automatically identifies the Pressure parameter and extracts the value 1.5.

The Concept of Data Grounding

Traditional AI might "guess" what a term means. Our system uses a technique called Data Grounding. Before the AI even attempts to extract values, it searches our Scientific Dictionary to find the exact, formal definitions that exist in our ontology. This ensures the AI is grounded in scientific reality, not statistics.

Higher Level Overview

Internal RAG Integration

Unlike other systems that use multiple disparate API calls, our "Fill-In" logic uses an Internal RAG (Retrieval Augmented Generation) loop powered by Discovery Engine.

1. The Grounding Search

The system first executes a semantic search to retrieve the most relevant DictionaryTerm objects based on the user's prompt.

2. The Augmented Extraction

The retrieved terms are passed as "context" to the internal model using a specialized PREAMBLE. This guides the model to return a structured JSON mapping.

// Example Instruction (Preamble)
ROLE: Data Extraction Assistant...
RULES:
1. KEY: Select the 'term' from the search results.
2. VALUE: Extract numeric/string value from QUERY.
FORMAT: {"Term": "Value"}

By combining search and summarization in a single atomic request, we maximize both speed and accuracy, following Google's generative search best practices.

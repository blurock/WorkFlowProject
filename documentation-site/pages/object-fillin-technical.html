Comparative Analysis

Building Block Architecture

The Value and Term Extraction is the most advanced application in our documentation suite. It is not a standalone tool, but rather a functional evolution that builds upon the foundations laid by RAG (Retrieval Augmented Generation) and Description Extraction.

Application Evolution

Concept Core Focus Key Technology
RAG Creating the Context. Mapping formal ontologies to physical data stores. Vector Search & SearchRequest Filters
Description Extraction Using Generative AI to summarize and distill natural language. SummarySpec & Narrative Preambles
Value and Term Extraction The Grounded Extraction which uses Context to drive Generative JSON output. ModelPromptSpec (Few-Shot Data Extraction)

Shared Tools & Divergent Uses

While all three systems share the Google Discovery Engine SDK and use the SearchServiceClient, they utilize the SearchRequest in fundamentally different ways:

  • Search Grounding: RAG and Fill-In both use .setFilter() to restrict the AI's "vision" to scientifically valid terms. Description Extraction typically operates on a broader context.
  • The Preamble Strategy: Extraction uses the Preamble to create readable prose. Fill-In uses a Strict Format Preamble to force the LLM into returning a machine-readable JSON mapping.
  • Data Synthesis: RAG returns existing data; Extraction generates new text; Fill-In performs a Dynamic Join between new user input and existing dictionary fields.
Implementation

The TermExtractionService Class

The core logic for extracting data models resides in the TermExtractionService. This service utilizes the SearchServiceClient to perform atomic search-and-summarize operations.

Data Model: ExtractedTerm

Each result is encapsulated in an ExtractedTerm object, which stores the relationship between the grounded dictionary entry and the extracted user value.

public class ExtractedTerm {
    private String term;           // Short name (e.g. "Enthalpy")
    private String namespacedTerm; // Full identifier (e.g. "dataset:Enthalpy")
    private String value;          // The extracted scientific value
    private JsonObject document;   // Full Firestore record context
    private double relevanceScore; // Search confidence
}
Prompting

Prompt Engineering (The Preamble)

The accuracy of the extraction depends on the PREAMBLE instructions. We use Few-Shot Learning by providing concrete examples within the prompt itself to ground the LLM's behavior. 

private static final String PREAMBLE = """
    ROLE: Data Extraction Assistant for scientific data.
    RULES:
    1. KEY: Select the 'term' from search results.
    2. VALUE: Extract numeric/string value directly from query.
    
    EXAMPLES:
    User Input: "temperature is 300 K"
    Model Output: {"TemperatureC": "300"}
    
    USER QUERY: {query}
    """;
API Usage

The SearchRequest Configuration

We configure the SearchRequest with a SummarySpec to trigger the internal summarization engine. 

SearchRequest request = SearchRequest.newBuilder()
    .setServingConfig(servingConfig)
    .setQuery(prompt)
    .setFilter(filter)
    .setContentSearchSpec(SearchRequest.ContentSearchSpec.newBuilder()
        .setSummarySpec(SearchRequest.ContentSearchSpec.SummarySpec.newBuilder()
            .setSummaryResultCount(maxResults) // Ground in top X results
            .setModelPromptSpec(
                SearchRequest.ContentSearchSpec.SummarySpec.ModelPromptSpec.newBuilder()
                    .setPreamble(PREAMBLE)
                    .build())
            .build())
        .build())
    .build();
Data Handling

Data Enrichment Logic

Because the LLM returns JSON while the search returns SearchResult objects, we must perform a Case-Insensitive Join to merge the extracted values back into our rich Firestore metadata.

Method: enrichWithValues

private List enrichWithValues(List<ExtractedTerm> terms, String llmJson) {
    JsonObject json = JsonParser.parseString(llmJson).getAsJsonObject();
    
    // Build a lowercase map for case-insensitive matching
    Map<String, String> valueMap = json.entrySet().stream()
        .collect(Collectors.toMap(
            e -> e.getKey().toLowerCase(),
            e -> e.getValue().getAsString()
        ));

    for (ExtractedTerm t : terms) {
        String val = valueMap.get(t.getTerm().toLowerCase());
        t.setValue(val != null ? val : "");
    }
}

Sanitization: We use rawJson.replaceAll("```json|```", "").trim() to handle potential markdown artifacts if the model returns a formatted code block.

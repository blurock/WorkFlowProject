Retrieval Augmented Generation (RAG)

Grounding Large Language Models in formal ontology through precise semantic retrieval.

Knowledge Base Search Grounding AI Generation

Overview

Effective data management requires mapping domain-specific terms (from the ontology) to physical datastore schemas. This system extracts dictionary terms from formal ontology classes and indexes them for high-performance semantic retrieval.

What is Retrieval-Augmented Generation?

Retrieval-Augmented Generation (RAG) is a technique that grants Large Language Models access to external, trusted data sources. Instead of relying solely on its training data, the model "retrieves" relevant information to "augment" its "generation" of responses, ensuring accuracy and grounding in specialized domains.

Technical Perspective

Storage & Semantic Search

The backend leverages two primary Google Cloud services to manage these terms:

  • Firestore: Used as the metadata repository to store the definitive list of terms and their relationships.
  • Discovery Engine (Data Store): Provides the semantic search layer (Vector Search) that allows for natural language matching beyond simple keyword lookups.

            .setServingConfig(servingConfig)
            .setQuery(query)
            .setFilter("metadata.datatype: ANY(\"" + datatype + "\")")

}
